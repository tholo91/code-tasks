# Tech-Spec: UX Polish Batch — Quick Fixes

**Created:** 2026-03-20

## Overview

### Problem Statement

Several small UX issues degrade the mobile experience: text gets selected during drag-reorder, bottom sheets don't close when tapping the backdrop, the PWA still shows stale icons on iOS, and the sync pill wastes space with "All caught up" text.

### Solution

A batch of 4 targeted, independent fixes — each touching 1-2 files, no architectural changes.

### Scope

**In Scope:**
1. Suppress text selection during long-press drag reorder
2. Fix bottom sheet backdrop tap to close (with pull-down animation)
3. Update PWA icons to new logo, add og:image meta tag, fix iOS cache staleness
4. Simplify sync status pill: remove "All caught up", show only checkmark + timestamp

**Out of Scope:**
- TaskCard flag icon + padding redesign (covered by Story 8-10)
- Sync result toast (covered by Story 8-6)
- Reorder button redesign (to be folded into Story 8-10)
- Umlaut support (needs separate investigation)

## Context for Development

### Codebase Patterns

- Motion config constants in `src/config/motion.ts` (`TRANSITION_SHEET`, `TRANSITION_SPRING`)
- `useReducedMotion()` from Framer Motion used throughout for a11y
- PWA manifest generated by `vite-plugin-pwa` in `vite.config.ts`
- Icons served from `/public/` directory
- Badge styling via `.badge` / `.badge-green` CSS classes in `index.css`

### Technical Decisions

- **PWA icons:** `maskable-icon-512x512-light.png` is the canonical app icon. All smaller sizes (64, 192, 180) should be generated from it. Old icon files (`pwa-64x64.png`, `pwa-192x192.png`, `pwa-512x512.png`, `apple-touch-icon-180x180.png`, `favicon.ico`) are to be replaced.
- **Bottom sheet animation:** Backdrop tap triggers same `onClose()` → exit animation `y: '100%'` with spring physics — identical to drag-to-close behavior. No separate animation needed.
- **Sync pill:** The checkmark icon in `SyncStatusIcon` uses a `strokeWidth`-equivalent via SVG path. To make it "fatter", increase the icon `size` prop from 10 to 12.

- [x] **Task 2: Fix bottom sheet backdrop close (Fix 2)**

**File:** `src/components/ui/BottomSheet.tsx`

**Bug:** The outer `motion.div` (line 29) has `onClick={(e) => { if (e.target === e.currentTarget) onClose() }}`. But the backdrop child `div` (line 44, `absolute inset-0`) intercepts clicks — `e.target` is the backdrop, not the outer div, so `e.target !== e.currentTarget` and `onClose()` never fires.

**Fix:** Add `onClick={onClose}` to the backdrop div (line 44-46). The sheet surface has `z-10` so clicks on the sheet won't hit the backdrop.

**Current code (line 44-46):**
```tsx
<div
  className={`absolute inset-0 bg-black/50${backdropBlur ? ' backdrop-blur-sm' : ''}`}
/>
```

**New code:**
```tsx
<div
  className={`absolute inset-0 bg-black/50${backdropBlur ? ' backdrop-blur-sm' : ''}`}
  onClick={onClose}
/>
```

The exit animation (`y: '100%'` with `TRANSITION_SHEET` spring) already provides the pull-down visual — same as drag-to-close. No additional animation work needed.

Also remove the now-redundant `e.target === e.currentTarget` check from the outer div's onClick (line 35-37), since the backdrop handles it. Replace with just preventing clicks on the outer div from doing anything unexpected — or remove the handler entirely since the backdrop covers the non-sheet area.

**Updated outer div (line 35-37):**
Remove `onClick` from the outer `motion.div` entirely — the backdrop div now handles closing.

- [x] **Task 3: Update PWA icons and add og:image (Fix 3)**

**Step 3a: Generate smaller icon sizes from new logo**

Using the `maskable-icon-512x512-light.png` as source, generate:
- `pwa-64x64.png` (64x64)
- `pwa-192x192.png` (192x192)
- `apple-touch-icon-180x180.png` (180x180)
Use `sips` (macOS built-in) to resize:
```bash
sips -z 192 192 public/maskable-icon-512x512-light.png --out public/pwa-192x192.png
sips -z 64 64 public/maskable-icon-512x512-light.png --out public/pwa-64x64.png
sips -z 180 180 public/maskable-icon-512x512-light.png --out public/apple-touch-icon-180x180.png
```
No `favicon.ico` needed — modern browsers accept `<link rel="icon" type="image/png">`. Point directly to `pwa-64x64.png` in `index.html`.

**Step 3c: Update `index.html`**

Replace existing icon links and add og:image meta tags with cache-bust query param:
```html
<link rel="icon" href="/code-tasks/pwa-64x64.png" type="image/png">
<link rel="apple-touch-icon" href="/code-tasks/apple-touch-icon-180x180.png?v=2">
<meta property="og:image" content="https://tbscode.github.io/code-tasks/maskable-icon-512x512-light.png" />
<meta property="og:image:width" content="512" />
<meta property="og:image:height" content="512" />
<meta property="og:title" content="Gitty" />
<meta property="og:description" content="Lightweight GitHub issue capture tool for developers on the go" />
```

Notes:
- og:image uses absolute URL (required by social crawlers — Twitter, Slack, iMessage won't resolve relative paths)
- `?v=2` on apple-touch-icon helps iOS Safari pick up the new icon on next PWA launch (not guaranteed, but free to try)
- `favicon.ico` link replaced with png reference — modern browsers handle this fine

**Step 3d: Clean up old files**

Delete `public/pwa-512x512.png` (replaced by maskable-icon-512x512-light.png), `public/favicon.ico` (replaced by pwa-64x64.png), and `public/vite.svg` (unused Vite placeholder).

### Acceptance Criteria

1. **Given** I long-press a task card to reorder, **when** I hold and drag, **then** no text selection highlight or iOS callout menu appears on the card content.

2. **Given** a bottom sheet is open, **when** I tap the dark backdrop area outside the sheet, **then** the sheet closes with the same pull-down spring animation as dragging it down.

3. **Given** I add Gitty to my iPhone home screen as a PWA, **when** the icon appears, **then** it shows the new logo (`maskable-icon-512x512-light.png`). **Given** I share the Gitty URL, **when** a social preview is generated, **then** the og:image shows the new logo.

4. **Given** the app is fully synced, **when** I look at the sync status pill in the header, **then** it shows a green checkmark icon (slightly larger than before) followed only by the relative timestamp (e.g., "2 min ago") — no "All caught up" text. **Given** the app has never synced (`lastSyncedAt` is null), **when** I look at the header, **then** no sync pill is shown.
